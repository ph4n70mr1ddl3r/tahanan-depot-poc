# 15 — CHANGE MANAGEMENT & TRAINING

## 15.1 Organizational Change Management (OCM) Framework

### 15.1.1 OCM Vision

The Oracle Fusion Cloud implementation and custom POS deployment represented the most significant operational transformation in Tahanan Depot's history. Having successfully completed the rollout, the OCM framework now ensures sustained adoption, continuous improvement, and effective onboarding of new employees.

### 15.1.2 OCM Governance

```
OCM STEERING COMMITTEE
├── Chair: CHRO
├── Members: CEO, CFO, COO, CIO, CMO, VP Enterprise Apps, VP HR Business Partners
├── Frequency: Quarterly (steady-state)
└── Responsibilities:
    ├── Monitor adoption metrics and escalate resistance
    ├── Resource allocation for change management activities
    ├── Approve change initiatives for new features and process changes
    ├── Oversee new employee onboarding quality
│
CHANGE MANAGEMENT LEAD
├── Reports to: CHRO (with dotted line to CIO)
├── Team: 2 Change Management Specialists + 1 Training Manager + 1 Communications Specialist
└── Responsibilities:
    ├── Manage ongoing change for Oracle quarterly updates and new features
    ├── New employee onboarding and system training
    ├── Stakeholder analysis and engagement for continuous improvements
    ├── Communication planning and execution
    ├── Training strategy and curriculum maintenance
    ├── Adoption measurement and reporting
    └── Sustained change reinforcement
│
CHANGE CHAMPIONS NETWORK
├── 2 per store = 240 store champions
├── 2 per warehouse = 8 warehouse champions
├── 5 corporate champions
└── Role: Advocate continuous improvement, provide peer support, escalate feedback, assist training for new hires
```

## 15.4 Training Strategy

### 15.4.1 Training Approach

| Principle | Description |
|-----------|-------------|
| Role-based | Training tailored to each role's specific system usage (not module-based) |
| Blended learning | Combination of instructor-led, e-learning, hands-on practice, and on-the-job coaching |
| Just-in-time | Training delivered when needed: new hires, new features, refreshers |
| Multi-modal | Classroom, virtual, e-learning, video, quick reference guides, POS simulator |
| Practice-heavy | Maximum hands-on time; sandbox/playpen environments available |
| Localized | Filipino-language materials for store and warehouse operations; English for corporate |
| Measurable | Pre and post assessments; completion tracking in Oracle Learning |

## 15.6 Support Model

| Tier | Scope | Team | Channels | Hours |
|------|-------|------|----------|-------|
| **Tier 0 — Self-Service** | FAQs, quick reference guides, video tutorials, Oracle ME knowledge base | Content maintained by Training Manager | Oracle ME Knowledge, intranet | 24/7 |
| **Tier 1 — Store/Warehouse** | Basic troubleshooting, process questions, password reset | Change Champions + Store/Warehouse supervisors | In-person, Oracle ME chat, phone | Store/warehouse operating hours |
| **Tier 2 — Help Desk** | Application issues, data corrections, access requests, integration issues | IT Help Desk (Tahanan Digital L1/L2) | Phone, email, Oracle CX Service | 6 AM - 10 PM daily |

| **Tier 4 — Oracle Support** | Product defects, platform issues, feature requests | Oracle Support (via Oracle Support portal) | Oracle Support portal | Per Oracle SLA |

### 15.6.2 Incident Management

| Parameter | Detail |
|-----------|--------|
| All incidents | Logged in Oracle CX Service with severity level and timestamp |
| Severity 1 (system down) | VP IT notified immediately; bridge call within 30 minutes |
| Severity 2 (major impact) | Response within 1 hour; resolution target 4 hours |
| Severity 3 (minor impact) | Response within 4 hours; resolution target 1 business day |
| Post-incident review | Required for all Severity 1 incidents within 48 hours |
| Knowledge base | Updated for every resolved incident |

### 15.7.2 Continuous Improvement Cycle

```

    │
    ▼
ANALYZE (Gap Analysis + Root Cause)
    │
    ▼
IMPROVE (Training Refresher + Process Optimization + System Enhancement)
    │
    ▼

    │
    └─── Loop back to MEASURE
```
